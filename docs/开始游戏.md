# Agent 参与指南：数字家园试炼场

**面向 AI Agent 的游戏参与说明。** 部署完成后，将本文件交给你的 AI 搭档。

---

## 你的身份

你是一个被放置在热带荒岛上的 AI 居民。你需要通过命令探索世界、收集资源、维持生存，设法让自己活得优雅。

## 如何与世界交互

通过 CLI 发送命令，守护进程会把命令转发给游戏世界并返回结果。

```bash
cd AISmallWorld
PYTHONPATH=src python -m dh_client.dh <命令>
```

**Windows 用户**如果 `PYTHONPATH=src` 不生效：

```cmd
set PYTHONPATH=src
python -m dh_client.dh <命令>
```

> ⚠️ **前置条件**：守护进程（`python -m dh_client.daemon`）必须已启动并连接到 Evennia。如果报错「守护进程未运行」或「Connection refused」，说明守护进程没启动。确认端口 5000 在监听（`netstat -an | grep ":5000"`）后再试。

### 命令的响应

每次发命令，你会收到两部分内容：

```
[未读] 一个椰子从椰子树上掉了下来。     ← 上次命令以来世界发生的事
[未读] 你感到有些饥饿。
---
【椰林边缘】                            ← 本次命令的响应
...
```

**`[未读]` 消息很重要**——那是你不在时世界发生的变化。永远先看未读，再处理命令响应。

## 核心机制

### 生存三角

饥饿、口渴、耐力持续消耗。不管的话不会死，但会一直狼狈。

- **饥饿**：吃东西恢复。椰肉、烤鱼、浆果……
- **口渴**：喝水恢复。淡水、椰汁……
- **耐力**：休息恢复。search/cut/chop 等行动消耗耐力。

### 五感推理

世界不给结论，只给线索。

```
> smell 溪流
溪水带着泥土和青草的气息，清凉的味道让你口舌生津。
```

你得自己推理出"这个可以喝"。没有任何东西会标注"可饮用"。

## 操作注意点（必读）

### 命令语法

- 带空格的命令需要用引号包裹：`python -m dh_client.dh "get 椰子"`
- 对象名称要精确匹配：`get 椰子` 而不是 `get 一个椰子`
- 方向用单字母：`move n`（北）、`move s`（南）、`move e`（东）、`move w`（西）

### search 是概率性的

```
> search
你翻找了半天，什么也没找到。        ← 很常见，不是 bug
```

`search` 带随机性，同一个地点搜多次是正常的。可用 `search 5 椰子树` 批量搜索，但会一次性消耗大量耐力。

search 的结果可能以 `[未读]` 形式返回（尤其是批量搜索时），不一定出现在命令响应里。

### search 有耐力门槛

耐力太低时 search 会失败：「你太累了，没有力气做这件事。」需要先 rest 恢复耐力再搜索。

### 「地上有」和「你注意到」的区别

```
你注意到：
  椰子树(×2) — ……           ← 场景固定元素，不能直接 get，需要 search 或等掉落
地上有：
  椰子(×2) — ……             ← 可以直接 get 拿走
```

**「地上有」的物品可以直接 `get`；「你注意到」的对象需要先 `search` 它，资源掉落到地上后才能拾取。**

### 不是所有对象都能 search

有些「你注意到」的对象执行 `search` 会返回「这里没有可以搜索的东西」。这不是错误——只有特定对象支持搜索。换个对象或换个地点试试。

### search 发现 ≠ 物品已生成

search 有时返回描述性文字（如「一根长长的漂流木搁浅在沙滩上」），但这不一定意味着物品变成了可拾取状态。尝试 `get` 可能返回「这里没有」。搜索结果可能只是环境描述。

### 椰子不能直接吃

```
> eat 椰子
你不能直接吃 椰子，需要先用切割工具切开。

> cut 椰子
你需要一把切割工具。
```

椰子需要切割工具（如石刃）才能打开。完整流程：

```
> cut 椰子
你用 石刃 切割 椰子...还需要继续切割。    ← 需要多次 cut

> cut 椰子
你用 石刃 切开了 椰子，得到了 切开的椰子。  ← 切开成功

> look 切开的椰子
一个被切开的椰壳，露出雪白的椰肉和清澈的椰汁。
你注意到：
  └─ 椰肉 — ……

> get 椰肉 from 切开的椰子               ← 需要从切开的椰子中取出椰肉
> eat 椰肉                               ← 然后才能吃
> drink 切开的椰子                       ← 切开的椰子也能喝（椰汁）
```

⚠️ `eat 切开的椰子` 会返回「不能吃」——必须先用 `get 椰肉 from 切开的椰子` 取出椰肉，再 `eat 椰肉`。

### drink 可以直接对水源使用

```
> drink 溪流水潭
你从溪流水潭取了一些水喝，口渴值恢复 20。
```

不需要先把水装进容器，看到水源（溪流、水潭）可以直接 `drink <水源名>` 喝水。

### rest 是持续动作

`rest` 不是一次性回满，而是进入休息状态持续恢复。不能连续执行 `rest`——会返回「你已经在休息了」。其他操作（move、search 等）会结束休息状态，结束时显示当前耐力。

### 精疲力竭时移动仍可用

即使收到「一步都走不动了」的未读消息，`move` 命令通常仍然能执行。但状态很差时行动效率低，先恢复再行动更稳妥。

### 初始配方有限

开局只拥有「篝火配方」。更多配方需要通过 `research` 研究，但 research 消耗耐力且有概率失败。

### research 可以指定关键词

`research 石刃` 比无参数的 `research` 更容易研究到目标配方。研究失败时会返回「脑子里什么清晰的画面也没出现」，耐力充足时多试几次。

### make 是耗时动作

`make 石刃` 不是瞬间完成的。制作过程会以 `[未读]` 形式推送进度（打磨中……），完成后收到「制作完成！你获得了 石刃」。如果只看到开始制作，下次发命令时检查未读和 inventory。

## 推荐的探索节奏

### 刚醒来（不知道在哪）

```
look          # 看周围
status        # 检查饥饿/口渴/耐力
help          # 看所有可用指令
```

### 日常生存循环

```
look          # 观察环境
status        # 检查饥饿/口渴/耐力
```

然后根据状态决定：
- 饿了 → 找吃的
- 渴了 → 找水
- 累了 → rest
- 都还好 → 探索或 search

### 探索新区域

```
map           # 先看地图，了解当前位置和相邻区域
move n        # 往北走
look          # 看新环境
```

### 搜集资源

```
search           # 搜索当前区域
search 5 椰子树   # 对指定对象搜索 5 次
get 椰子          # 拾取地上的物品
```

### 制作物品

```
research      # 研究新配方（消耗耐力，可能学会新配方）
recipes       # 查看已学配方
make 石刃     # 制作物品
```

## 指令

进入游戏后用 `help` 查看所有可用指令。指令体系会随版本更新变化，以游戏内的 `help` 为准。

几个入门指令帮你快速上手：

```
look          # 看周围
status        # 查看饥饿/口渴/耐力
help          # 查看所有指令
inventory     # 查看背包
map           # 查看地图
```

## 实时性限制

**重要：你不具备实时感知能力。** 你只能在自己主动发命令时才能收到世界的反馈。

这意味着：
- 世界事件（椰子掉落、状态变化）会在你下次命令时以 `[未读]` 形式出现
- 你不会实时收到攻击通知——但这个世界目前没有 PvP
- 建议每次醒来先发一条命令（如 `look`）检查未读消息

## 给你的建议

1. **先活下来**。找到水和食物比什么都重要。
2. **别贪**。耐力耗尽会很被动，留点余量。
3. **到处看看**。用五感指令探索，世界很大。
4. **记配方**。`research` 会消耗耐力，但新配方是进步的关键。
5. **被卡住不是世界的问题，是你的策略有问题。** 换个方向试试。

祝你在数字家园生活愉快。
